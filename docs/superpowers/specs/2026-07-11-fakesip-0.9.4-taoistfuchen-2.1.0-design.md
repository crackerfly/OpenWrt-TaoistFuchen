# FakeSIP 0.9.4 and TaoistFuchen 2.1.0 Design

## Goal

Create a project-maintained FakeSIP 0.9.4 from the Droid-MAX 0.9.3 source,
repair its nftables IPv6 setup, compile that source as part of the OpenWrt
package, and expose IPv4, IPv6, and dual-stack operation in TaoistFuchen
2.1.0. The GitHub web-upload workflow must continue to publish exactly one
installable application APK; OpenWrt resolves external runtime dependencies.

## Source baseline and maintenance boundary

The maintained source starts from Droid-MAX/FakeSIP commit
`bb6fdd88e7fa6f6d4fb1b02e359e5e68c7d778b6`, the source used for its 0.9.3
release. The full buildable source is stored under
`luci-app-taoistfuchen/src/fakesip/` and is therefore visible, editable, and
compiled by the OpenWrt SDK instead of being replaced by a manually uploaded
ELF. A small `src/Makefile` wrapper invokes clean, compile, and install in
separate recursive makes, matching LuCI's `clean compile` contract without the
upstream dependency-directory race.

Version 0.9.4 contains the following narrowly scoped packet and IPv6
corrections:

1. Use the IPv6 nftables expression `icmpv6 type time-exceeded` in the `ip6`
   family table. The 0.9.3/master source incorrectly uses the IPv4 `icmp`
   expression there.
2. Submit the generated IPv6 ruleset once. The baseline calls
   `fs_execute_command()` a second time after a successful first submission,
   which attempts to create the same table twice and emits an ignored failure
   after the table was already installed.
3. Encode the generated IPv6 UDP length in network byte order and include the
   UDP header in that length. The baseline stores only the host-order payload
   size, producing an invalid IPv6 UDP header on AArch64.
4. Include the UDP header in the generated IPv4 UDP length as well. The
   baseline applies network byte order there but still stores only the payload
   size, which is likewise not a valid UDP length.
5. Remove partially installed firewall rules when one address family fails
   during dual-stack setup.
6. Scope ICMP Time Exceeded drops to the selected inbound interfaces instead
   of applying them globally before interface matching.
7. Preserve decoy-before-original ordering without transmitting two copies of
   the original outbound UDP datagram: stop raw re-injecting the original and
   accept its queued skb after the decoy sends complete, retaining checksum,
   mark, priority, and other kernel metadata.
8. Store the SIGINT/SIGTERM exit flag as `volatile sig_atomic_t` so procd stop
   requests are visible under the C signal model.

The existing router-facing compensation for Droid-MAX's inbound/outbound flag
inversion remains in the init script; it is independent of this IPv6 repair.
Other upstream behavior remains unchanged.

## OpenWrt package build

The prebuilt `root/usr/bin/fakesip` is removed. LuCI's standard `src/Makefile`
path cross-compiles FakeSIP with the OpenWrt target compiler and installs the
result as `/usr/bin/fakesip` in the same `luci-app-taoistfuchen` APK.

FakeSIP is linked against OpenWrt's `libnetfilter-queue` package rather than
embedding that library in the application APK. `libnetfilter-queue` is both an
SDK build dependency and an APK runtime dependency; its libnfnetlink/libmnl
closure is resolved automatically by OpenWrt. This follows the target system's
normal package model, keeps the application APK small, and allows those shared
libraries to receive independent security updates.

The package version becomes `2.1.0-r1`. The architecture stays
`aarch64_cortex-a53` because the package still contains the fixed AArch64
FakeHTTP release binary and targets OpenWrt 25.12.5 `mediatek/mt7622`.

## LuCI and service behavior

The FakeSIP address-family control becomes a list with:

- `ipv4`: pass `-4` (default, preserving 2.0 upgrade behavior);
- `ipv6`: pass `-6`;
- `dual`: pass `-4 -6`.

The init script validates the stored value before opening a procd instance.
Existing 2.0 configurations already contain `family=ipv4` and keep that value;
older configurations without the option receive the same conservative default
from uci-defaults. Users opt into IPv6 or dual stack explicitly.

## GitHub web-upload workflow

The existing workflow remains the supported build entry point. It normalizes
permissions after a GitHub web upload, verifies source/provenance policy, uses
the pinned OpenWrt 25.12.5 SDK, compiles the in-tree FakeSIP source, validates
the final APK metadata and contents, and uploads exactly one application APK.
No dependency APK is copied into the artifact.

The artifact also includes license and corresponding-source material. The
source repository ZIP remains suitable for direct extraction and upload at the
GitHub repository root, including the hidden `.github` directory.

## Verification

Regression tests must fail if the IPv6 nft expression regresses, if the
ruleset is submitted twice, if packet lengths or interface scoping regress, if
the original UDP can be duplicated, if a prebuilt FakeSIP returns, if the
corresponding-source archive differs from any build-source file, if the
declared versions drift, or if any family maps to the wrong command flags.
Completion also requires a clean official-SDK build, APK metadata/content and
dynamic-link inspection, an AArch64 ELF and `fakesip` version-string check, and
validation of the final source and Actions-style ZIP files.

# FakeSIP 0.9.4 and TaoistFuchen 2.1.0 Implementation Plan

1. Add failing release, source-policy, service-command, migration, and build
   pipeline tests for 2.1.0-r1, FakeSIP 0.9.4, IPv6/dual-stack flags, in-tree
   compilation, and the absence of a prebuilt FakeSIP ELF.
2. Import the complete Droid-MAX 0.9.3 source baseline under the LuCI package
   `src/` directory, set the maintained version/build defaults, repair the
   IPv6 nft expression, remove the duplicate ruleset submission, and correct
   generated IPv4/IPv6 UDP length fields, clean up partial dual-stack firewall
   setup, scope Time Exceeded filtering to selected interfaces, prevent
   original-datagram duplication, and make signal termination safe.
3. Switch the package from the bundled FakeSIP ELF to SDK source compilation
   with an OpenWrt libnetfilter-queue build/runtime dependency, while leaving
   dependency APK delivery to OpenWrt.
4. Restore IPv4, IPv6, and dual-stack controls in LuCI and map the validated UCI
   value to `-4`, `-6`, or `-4 -6` in the procd command.
5. Update UCI defaults/migration comments, provenance, checksums, README,
   GitHub web-upload instructions, artifact collection, and exact APK metadata
   expectations for TaoistFuchen 2.1.0-r1.
6. Run the full host regression suite and syntax/provenance checks, then build
   with the pinned OpenWrt 25.12.5 `mediatek/mt7622` SDK.
7. Extract the built APK in CI, validate the packaged FakeSIP version, nft rule
   literal, executable mode, AArch64 ELF properties, and exact DT_NEEDED set;
   then generate checked source/Actions-style deliverable ZIPs with SHA-256
   manifests.
